# 도서관 관리 시스템 (Library Management System)

## 프로젝트 개요

본 프로젝트는 C++로 구현한 콘솔 기반 도서관 관리 시스템입니다.  
도서 등록·검색·대출·반납 등 도서관 운영에 필요한 핵심 기능을 객체지향 설계 원칙에 따라 구현하며,  
3인이 기능 단위로 역할을 분담하여 협업 개발합니다.

### 주요 목표

- 실제 도서관 업무 흐름을 소프트웨어로 구현
- C++ 클래스 설계, 상속, 파일 입출력 등 핵심 개념 실습
- Git을 활용한 팀 협업 및 버전 관리 경험

## 기능 상세 소개

### 도서 관리 `src/book/` | 난이도 ⭐⭐⭐

도서관의 핵심 데이터인 **도서 정보를 관리**하는 모듈입니다.  
도서 데이터를 파일로 영구 저장·불러오기까지 담당합니다.

**기본 구현 항목**
- 도서 클래스 설계 (ISBN, 제목, 저자, 출판사, 출판연도, 재고 수량)
- 도서 등록 — 새 도서를 입력받아 목록에 추가, 도서 ID 자동 부여
- 도서 수정 — 기존 도서 정보(제목, 저자 등) 변경
- 도서 삭제 — 대출 중인 도서는 삭제 불가 처리
- 도서 목록 전체 출력 — 테이블 형식으로 콘솔 출력
- 파일 입출력 — 도서 데이터 읽기/쓰기

**추가 구현 항목 — 카테고리 분류**
- 도서에 카테고리 속성 추가 (소설, 과학, 역사, 기술 등)
- 카테고리별 도서 목록 필터 출력
- 카테고리 기준 정렬 기능

**핵심 고려사항**
- 재고 수량이 0일 때 대출 불가 상태 표시
- 도서 ID 중복 방지 및 자동 증가 처리
- 카테고리는 상수로 통일 관리

---

### 회원 관리 `src/member/` | 난이도 ⭐⭐⭐

도서관을 이용하는 **회원 정보를 관리**하는 모듈입니다.  
회원 데이터를 파일로 저장·관리합니다.

**기본 구현 항목**
- 회원 클래스 설계 (회원 ID, 이름, 연락처, 가입일, 대출 가능 여부)
- 회원 가입 — 회원 정보 입력 및 ID 자동 부여
- 회원 정보 수정 — 이름, 연락처 변경
- 회원 탈퇴 — 대출 중인 도서가 있을 경우 탈퇴 불가 처리
- 회원 목록 전체 출력
- 파일 입출력 — 회원 데이터 읽기/쓰기

**추가 구현 항목 — 연체 벌금 계산**
- 회원 정보에 누적 벌금 금액 추가
- 반납 시 연체 일수 × 단위금액(100원) 자동 계산 후 누적
- 벌금 납부 처리 — 납부 시 누적 벌금 차감
- 벌금 미납 시 추가 대출 불가 처리

**핵심 고려사항**
- 대출 중 회원 탈퇴 방지 로직
- 벌금 미납 상태와 대출 가능 여부 연동
- 대출·반납 담당자와 벌금 계산 로직 연동 필요

### 대출·반납 + 검색·통계 `src/loan/` `src/search/` | 난이도 ⭐⭐⭐⭐

시스템의 **핵심 업무 로직과 데이터 조회**를 담당하는 모듈입니다.  
도서 재고, 회원 상태, 날짜 계산이 모두 연동되며, 전체 데이터를 집계·출력합니다.

**대출·반납 구현 항목**
- 대출 클래스 설계 (대출 ID, 회원 ID, 도서 ID, 대출일, 반납 예정일, 반납일)
- 대출 처리 — 재고 확인 → 회원 대출 가능 여부 확인 → 대출 등록 → 재고 차감
- 반납 처리 — 반납일 기록, 재고 복구, 연체 여부 확인
- 연체 계산 — 현재 날짜와 반납 예정일 비교, 연체 일수 산출
- 대출 이력 조회 — 특정 회원의 대출 기록 출력
- 파일 입출력 — 대출 데이터 읽기/쓰기

**검색·통계 구현 항목**
- 도서 검색 — 제목, 저자, ISBN 키워드 검색 (부분 일치 지원)
- 회원 검색 — 이름, 회원 ID로 검색
- 대출 현황 통계 출력
  - 현재 대출 중인 도서 수 / 반납 완료 수
  - 연체 중인 도서 목록 및 연체 일수
- 전체 현황 요약 대시보드 출력

**핵심 고려사항**
- C++ `<ctime>` 또는 `<chrono>`를 활용한 날짜 처리
- 대출 가능 권수 제한 (예: 최대 3권) 처리
- 도서·회원 모듈과의 데이터 연동 필요
- 대소문자 구분 없는 문자열 검색 처리

## .h 와 .cpp 의 관계

C++에서 클래스는 **선언부**와 **구현부**로 분리됩니다.

| 파일 | 역할 | 
|------|------|
| `.h` (헤더) | 클래스 이름, 멤버 변수, 함수 목록 선언
| `.cpp` | 함수의 실제 동작 구현

```
Book.h                          Book.cpp
─────────────────────           ─────────────────────────────
class Book {                    bool Book::isAvailable() {
    int stock;                      return stock > 0;    ← 실제 동작
    bool isAvailable();  ──────►}
};               선언만
```

### 팀원 간 협업에서의 역할

`.h` 파일은 팀원 간 **계약서** 역할을 합니다.

도서 담당자의 `Book.cpp` 구현이 완성되지 않아도 `Book.h`만 보고 대출·반납 담당자가 코드를 미리 작성할 수 있습니다.

```cpp
// 대출·반납 담당자가 LoanManager.cpp 작성 시
#include "../book/Book.h"    // .h만 있으면 함수 목록을 알 수 있음

bool LoanManager::loanBook(int memberId, int bookId) {
    Book* book = ...
    if (!book->isAvailable()) return false;  // Book.cpp 완성 전에도 작성 가능
    book->decreaseStock();
}
```

### 개발 순서 권장

```
1단계  각자 .h 파일 작성 → 팀원들과 함수명/매개변수 합의
2단계  .h 기준으로 각자 .cpp 구현 (병렬 작업 가능)
3단계  main.cpp에서 통합 후 테스트
```

> `.h`가 먼저 확정되어야 다른 팀원이 기다리지 않고 동시에 개발할 수 있습니다.

```

├── src/
│   ├── book/         # 도서 관련 클래스

```
